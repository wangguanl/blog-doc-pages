自我介绍 #

各位领导、各位同事，大家上午好，我是前端开发工程师 - 王刚，今天为大家做个人述职报告，那我就直奔主题。在两个月的努力下做多个项目的工作。在完成紧张的排期需求，填了很多坑，还额外做了以下工作，我是做技术的，所以更多的是从技术层面出发。

几年下积累的问题，没有规范，无规律，无法评估完善时间，需要调研然后评估时间。

工作大事件 #

  1. 原创小程序（微信、百度、头条）

    • 业务层面

      • 用户手机号一键授权登录
      • 对项目进行分包，通过组件化提升维护性，对全局业务组件进行封装，本地静态资源上传至 CDN 提高加载速度，总包由 2.1MB 压缩至 1.4MB，体积降低了66%，性能评估报告由 **43%**提升至 96%，小程序 首次 打开访问速度，由 3~5 秒减少至 1~2 秒

    • 架构优化层面

      • 历史代码的质量和逻辑严谨上有所欠缺，对三端进行了代码合并改版，做到了一套代码产出标准化三端（微信、百度、头条）
      • 改进微信端的发版上线方式，将大量重复人工上线发版的问题进行工程化处理

  2. 原创 App

    • 环境拆分（本地开发环境、测试环境、预生产环境、正式生产环境）
      • 改善了项目开发阶段的调试问题，为测试环境提供了调试查错功能
      • 通过工程化区分打包环境，杜绝打包环境不匹配的低级错误，提升了上线速度和稳定性
      • 配合运维搭建 Jenkins 一键上线部署环境，提升了开发阶段调试效率
    • 新增 vConsole 控制台，快速定位问题，提高修复 bug 效率

  3. 专题页项目

    1. 专题页批量替换当前页美洽弹出方式、批量增加 gio 提交表单埋点

      • 问题

        • 代码无规范、无规则，无法通过程序化大批量处理专题页
        • 验收人员验收时不严谨，导致专题页本身存在问题就发布上线
        • 非技术人员修改专题页代码，使专题页代码结构、脚本等出错，页面宕机，后续内容不加载，导致程序无法正常运行

      • 解决方案

        • 严格验收专题页
        • 通过后台系统给非技术人员提供可配置修改项
        • 建立代码规范
    2. 低代码、自建站等工具调研

目标计划 #

  1. 对 **原创小程序 **和 原创 App 做了大量的业务逻辑更替和项目架构底层改版，使得焕然一新。但还不够完善，需要通过技术层面一一改进、重构。
  2. **官网 **不符合前端开发架构，无预编译工具，本地开发效率低、调试效率低。上线无打包编译，代码冗余、结构混乱、文件目录结构不规范，上线出错率高，线上体验感待提高；依赖后端环境，环境问题层出不穷。
项目隐患： #
  • 整体结构逐渐衰弱
  • 代码质量不断降低
  • 代码无可读性
  • 弹性越来越差
  • 冗余代码越来越多
  • 新的需求只能胡乱拼凑上
  • 老旧代码无人能懂更不敢随意优化，稍不注意就会导致意想不到的灾难发生
  • ......

重构方案

  • 小程序：uni-app、Vue3、TypeScript
  • 官网：SSR、Nuxt、Webpack、Node、Vue、TypeScript
  • 官网后台：Vite、Vue3、TypeScript、Element-Plus
  • 重构方案 - 根据用户浏览器访问数据统计、大厂处理非兼容规避方案进行整理了两套重构方案（向下兼容：Webpack、jquery）

重构收益

  • 提高程序稳定性
  • 提高开发的速度 && 方便定位错误
  • 提高程序可控性
  • 提高程序性能
  • 提升用户操作体验
  • 定期Code Review，发现并解决问题

“戴商业的眼镜”来写项目，创造更大价值 #

  1. 通过技术给公司带来持续的竞争力
  2. 预研：为将来的发展作铺垫，做出别人不能做出的产品
  3. 完善埋点系统，供运营去分析数据，持续改进加强用户体验。
  4. 提高企业的生产力

2021-规划 #

  1. 保障业务团队的流畅开发
  2. 完善专题页项目体系
  3. 专题项目的相关培训
  4. 绘制重构蓝图

2022-规划 #

  1. 提升技术：团队士气最重要，技术不断更新，过度具象化不可取
  2. 团队规范标准化：工作流规范、编码规范、文档规范、UI设计规范、测试规范、前后端协作规范
  3. **工程标准化 **
    • 统一脚手架
    • 完善监控报警系统，快速定位发现线上问题。
    • 完善版本管理系统，自动构建工具，脚本。
  4. 项目重构
  5. 角色转变：晋升前端Leader
